## 8. 測試與驗證過程

先設定 `gradlew` 可執行：

```bash
chmod +x /Users/toast/employee-seating-system/gradlew
```

第一次執行：

```bash
./gradlew test
```

因 sandbox 無法直接連 Docker socket，出現：

```text
permission denied while trying to connect to the docker API
```

改以授權方式執行後，Docker 拉取 `gradle:8.10.2-jdk17`。

第一次測試結果發現 root project 的 `./gradlew test` 沒有帶到 `backend:test`，因此修改 root `build.gradle`，讓 `test` 任務依賴：

```gradle
tasks.register('test') {
    dependsOn ':backend:test'
}
```

再次測試發現 DTO 驗證失敗：

```text
AssignmentValidationTest > should_reject_invalid_employee_ids_when_validating_dto() FAILED
```

原因：

- `@Pattern` 不會拒絕 `null`
- 題目要求 null 員編要被視為 400

修正：

```java
@NotBlank @Pattern(regexp = "^\\d{5}$") String empId
```

修正後再次執行：

```bash
./gradlew test
```

結果：

```text
BUILD SUCCESSFUL
```

## 9. Docker Compose 驗證過程

執行：

```bash
docker compose up -d --build && curl -f http://localhost/api/seats
```

第一次 build 失敗：

```text
eclipse-temurin:17-jre-alpine: no match for platform in manifest
```

原因：

- `eclipse-temurin:17-jre-alpine` 對目前平台沒有可用 manifest。

修正：

```dockerfile
FROM eclipse-temurin:17-jre
```

同時補上：

```text
frontend/vite.config.js
```

並修正 PostgreSQL JSON 欄位大小寫解析，將：

```sql
jsonb_to_recordset(p_changes) AS x(empId TEXT, toSeatSeq INT)
```

改為：

```sql
jsonb_to_recordset(p_changes) AS x("empId" TEXT, "toSeatSeq" INT)
```

重新執行 Docker Compose 後，第一次 `curl` 出現：

```text
502
```

查看 logs 後確認：

- PostgreSQL 初始化成功
- Spring Boot 啟動成功
- 502 是 app 尚未完全 ready 時立即 curl 造成

稍後重試：

```bash
curl -i http://localhost/api/seats
```

結果：

```text
HTTP/1.1 200
```

並成功取得座位資料。

## 11. PUT 交易路徑驗證與修正

第一次驗證 PUT：

```bash
curl -i -X PUT http://localhost/api/seats/assignments \
  -H 'Content-Type: application/json' \
  -d '{"snapshotVersion":"0","changes":[{"empId":"10001","toSeatSeq":1}]}'
```

結果：

```text
HTTP/1.1 500
```

查看 app logs 發現：

```text
ERROR: column x.emp_id does not exist
Hint: Perhaps you meant to reference the column "s.emp_id".
```

原因：

- Stored Procedure 中 `INSERT INTO tmp_changes` 使用了錯誤 alias。

修正：

```sql
SELECT s.emp_id, s.to_seat_seq
```

修正後重建 DB volume：

```bash
docker compose down -v
docker compose up -d --build
```

再次驗證 PUT：

```bash
curl -i -X PUT http://localhost/api/seats/assignments \
  -H 'Content-Type: application/json' \
  -d '{"snapshotVersion":"0","changes":[{"empId":"10001","toSeatSeq":1}]}'
```

結果：

```text
HTTP/1.1 200
```

確認：

- `10001` 成功指派到座位 1
- `snapshotVersion` 從 `"0"` 變為 `"1"`
- 代表座位更新與 log 寫入成功
